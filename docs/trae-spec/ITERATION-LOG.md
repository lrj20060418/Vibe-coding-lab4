# /spec 迭代修正记录（全局）

> 记录开发过程中发现的需求模糊点及修正过程，符合实验「先修正 Spec，再改代码」要求。

## 迭代 #1：日历展示范围

**发现阶段**：Spec 2 设计 / 实现 UnifiedCalendar 前

| | 内容 |
|---|------|
| **初版（模糊）** | 「日历展示学习计划」 |
| **问题** | 未说明是单个计划还是全部计划；实现时可能只做当前计划日历 |
| **修正后（明确）** | 「日历展示**所有已保存计划**的任务，按日期**聚合合并**，格子显示合计时长或任务数」 |
| **代码影响** | `calendarAggregate.ts` 遍历全部 plan；`UnifiedCalendar` 接收 `plans[]` 而非单个 planId |
| **验收** | 两计划 6月2日均有任务 → 日历仅一格，显示合计 3h ✅ |

## 迭代 #2：每日详情组织方式

**发现阶段**：Spec 3 设计 / 路由设计前

| | 内容 |
|---|------|
| **初版（模糊）** | 「查看某天详情」 |
| **问题** | 可能设计成 `/plan/:planId/day/:index`，只能看单个计划的一天 |
| **修正后（明确）** | 「以**日期**为中心展示当天**所有计划**的任务；路由 `/day/:date`；禁止 planId + dayIndex」 |
| **代码影响** | `App.tsx` 路由 `/day/:date`；`getDayPlanEntries(plans, date)` 聚合多计划 |
| **验收** | 6月2日详情页同时显示 React + 线性代数任务 ✅ |

## 迭代 #3：AI API 地址配置

**发现阶段**：Spec 4 联调 / 接入智谱 API 时

| | 内容 |
|---|------|
| **初版（模糊）** | 「前端调用 OpenAI API」 |
| **问题** | API Key 暴露在前端；用户将 `AI_API_BASE` 填为完整 endpoint 导致 404 |
| **修正后（明确）** | 「Express 后端代理；Key 仅存 `.env`；`AI_API_BASE` 只填根地址如 `https://open.bigmodel.cn/api/paas/v4`」 |
| **代码影响** | 新增 `server/`；Vite proxy `/api`；`normalizeBaseUrl` 防重复路径 |
| **验收** | `/api/ai/status` configured=true；大模型标签正常 ✅ |

## 迭代 #4：AI 模型名默认值

**发现阶段**：智谱 API 联调

| | 内容 |
|---|------|
| **初版** | 未设置 `AI_MODEL`，默认 `gpt-4o-mini` |
| **问题** | 智谱返回 400「模型不存在」 |
| **修正后** | `.env` 显式设置 `AI_MODEL=glm-4-flash`；文档补充各平台模型名 |
| **验收** | AI 建议与细化可正常返回 ✅ |

## 迭代 #5：Markdown 展示方式

**发现阶段**：每日细化 UI 验收

| | 内容 |
|---|------|
| **初版** | `<pre>` 原样展示 Markdown 字符串 |
| **问题** | 用户看到 `#`、`*` 等符号，不符合「Markdown 展示」预期 |
| **修正后** | 使用 `react-markdown` 组件 `MarkdownContent` 渲染 |
| **验收** | 标题、列表、引用正确格式化 ✅ |
