# Lab 4 实验总结

## 1. 你的系统分为哪些模块？

我把「学习计划工作台」拆成几块，每块只做一件事：

| 部分 | 干什么用 |
|------|----------|
| **首页** | 创建计划、看整体进度、管理历史计划 |
| **计划表单** | 填学习主题、天数、每天学多久，并检查有没有填错 |
| **历史计划列表** | 看以前建过的计划，可以导出或删除 |
| **统一日历** | 一张月历里显示**所有计划**的安排 |
| **每日预览** | 点日历某一天，弹窗快速看当天任务 |
| **每日详情页** | 看某一天的完整任务，可以勾选「已完成」 |
| **AI 学习建议** | 根据现有计划给提醒，比如哪天学太多、进度如何 |
| **每日计划细化** | 把当天任务展开成更细的学习步骤 |
| **本地存储** | 用浏览器保存计划和勾选状态，刷新不丢 |
| **AI 小后端** | 单独跑一个服务去调大模型，密钥不写在网页里 |

用起来就是：建计划 → 在日历上看 → 点某天进详情 → 勾选完成 → 首页看进度。

## 2. 你的学习计划数据结构是如何设计的？

可以想成「三层嵌套」：

```
一份学习计划
  └── 很多「某一天的安排」
        └── 每天下面有多个「具体任务」
```

- **计划**：名字、从哪天开始、一共几天、每天几小时、备注等  
- **某一天的安排**：具体日期、当天目标、下面有哪些任务  
- **任务**：标题、说明、预计学多久、练习建议、有没有做完  

另外几点：

- 可以同时有多份计划（比如 React 和线性代数一起学）  
- 同一天可以来自不同计划（6 月 2 日两个计划都有任务）  
- 每个任务可以单独标记完成或未完成  

AI 细化出来的内容另外存：一段 Markdown 文字 + 几条可勾选的「完成标准」。

## 3. 你如何实现「统一日历」？

思路很简单：**把所有计划里、同一天的安排加在一起，日历上只占一个格子。**

做法：

1. 读出所有已保存的计划  
2. 按「日期」把任务归到一起  
3. 如果同一天有多个计划，把**时长和任务数加在一起**  
4. 格子里显示类似「3 小时 · 4 项任务」  

这样不用在好几个日历之间切换，打开首页就知道「这天所有计划加起来要学什么」。

点有任务的日期会先弹预览；想仔细做任务或勾选，再进详情页。

## 4. 你如何保证每日详情是以日期为中心的？

我没有用「某个计划的第几天」这种方式，而是：

- 网址写成 **`/day/2026-06-02`** 这种「以日期为主」的形式  
- 打开 6 月 2 日时，会把**当天所有计划**的任务都找出来一起显示  
- 页面上按 plan 分组，比如上面「React 基础」，下面「线性代数复习」  

因为用户真正关心的是 **「今天我要做什么」**，不是「React 计划第 3 天是什么」。如果 6 月 2 日两个计划都有课，应该全部列出来，而不是只能看其中一个。

## 5. 你在 /spec 模式中发现或修正过哪些需求问题？

实验要求用 Trae 的 /spec；我实际用 **Cursor** 做了同样的事：先把需求写清楚，再写代码，不对就回去改需求。

印象最深的几次：

| 一开始（比较模糊） | 后来改清楚 |
|-------------------|-----------|
| 「日历展示学习计划」 | 要展示**所有计划**，同一天要**合并** |
| 「查看某天详情」 | 以**日期**为主，看当天**全部计划**的任务 |
| 「网页里直接调 AI」 | 改成小后端调，密钥放 `.env` |
| AI 地址填错，报 404 | 地址只填根路径，不要多写 `/chat/completions` |
| 模型名没配，报 400 | 智谱要用 `glm-4-flash`，不能用默认的 gpt |
| 细化结果满屏 `#` 符号 | 改成正常渲染 Markdown，标题列表能看懂 |

更完整的修正过程见 `docs/trae-spec/ITERATION-LOG.md`。

## 6. 自动化测试、MCP 辅助开发、AI 增强或自定义 Agent 中，你尝试了哪些？效果如何？

### 自动化测试 —— 做了

- 用 **Vitest** 写了 34 个小测试，命令 `npm test` 都能过  
- 测了：表单填错会不会拦、计划能不能保存、日历会不会合并、详情页对不对、勾选会不会丢、整条流程走通不走通  
- **效果**：改代码后心里踏实一点，和 Spec 里写的场景也能对上  

### MCP 辅助开发 —— 试过

- 写界面时用 **HeroUI 的 MCP** 查组件怎么用  
- **效果**：主要是查文档方便，没有做到产品里给用户用  

### AI 增强 —— 做了（学习建议 + 每日细化）

**首页 AI 建议**

- 调智谱大模型，根据现有计划给本周概览、哪天学太多、进度怎么样  
- 如果 API 不通，会自动用本地简单规则，页面上也会提示  
- **效果**：配好 Key 后能看到「大模型」字样，建议比较具体  

**详情页「细化当天计划」**

- 点按钮后 AI 生成学习材料、练习、完成标准，用 Markdown 展示  
- 完成标准可以勾选，刷新还在  
- **效果**：比死板的模板灵活；后来加了 Markdown 渲染，看起来正常  

### 自定义 Agent —— 没做

- 没有做带「上网搜索、读网页」那种多步 Agent  
- 时间有限，而且说明书写这是选做；普通 API 调用已经够完成 AI 建议和细化  

### Cursor 协作 —— 做了

- 用 Cursor 拆需求、分步写代码、联调、写测试  
- 对话导出在 `docs/trae-chat/`  
- **效果**：可以一阶段一阶段做，每步都能跑起来，比一次写完再修轻松
